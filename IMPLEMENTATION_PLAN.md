# Simple Image Panel - Local Execution Plan

## Goal

Build a local personal image generation page focused on text-to-image and image-to-image workflows. The first version targets only `model: "gpt-image-2"` and follows the PackyAPI GPTImage Images API style:

- Text-to-image: `POST {baseUrl}/v1/images/generations`
- Image-to-image: `POST {baseUrl}/v1/images/edits`
- The API supports `n = 1`, so multiple requested images are generated by serial requests.

## Confirmed Requirements

- Build on `C:\code\image-simple-panel`.
- Local personal use.
- Use a local frontend plus local backend proxy.
- Provider settings include provider name, API key, and Base URL.
- Providers can be added, edited, deleted, and selected from a dropdown.
- API keys are encrypted at rest with a fixed default development secret.
- API key fields are hidden by default and can be temporarily shown.
- Main layout follows the provided dark three-column reference:
  - Left sidebar: providers, upload/input images, model and generation parameters.
  - Main area: generated result preview and prompt box.
  - Right sidebar: thumbnail list for generated images in this program.
- "Learn more" is replaced by provider management entry.
- Text-to-image and image-to-image modes are selectable.
- Image-to-image supports multiple uploaded input images.
- Uploaded input images are copied into the program workspace and renamed as `image_1`, `image_2`, etc.
- Input image delete only deletes the program copy, not the user's original file.
- Image-to-image uses all uploaded input images by default.
- Input image limit is 16 images; show a user-facing prompt when the limit is reached.
- Generated image count greater than 1 is handled by serial backend requests.
- Failed requests show the user the API error message and provide retry.
- Generated images are cached locally first.
- Clicking the page's download action marks a generated image as downloaded.
- On next backend startup, generated images that were not marked downloaded are cleaned up.
- Downloaded generated images remain in local cache and can still be previewed from history.
- Browser refresh keeps generated history visible.
- Main image right-click menu includes:
  - Copy source URL
  - Download image
  - Copy local image path
- Initial parameter support:
  - `model`: fixed `gpt-image-2`
  - size: `auto`, `1024x1024`, `1536x1024`, `1024x1536`, `2048x2048`, `2048x1152`, `3840x2160`, `2160x3840`
  - quality: `auto`, `low`, `medium`, `high`
  - moderation: `auto` or `low`, default `auto`
  - generation count
  - background: fixed `auto`, no UI
  - output format: fixed `png`

## Recommended Technical Stack

- Frontend: Vite, React, TypeScript.
- Styling: plain CSS with design tokens and responsive layout.
- Icons: lucide-react.
- Backend: Node.js, Express.
- Upload handling: multer.
- HTTP calls: native `fetch` and `FormData` in Node runtime.
- Local persistence: JSON files under `data/`.
- Binary file storage:
  - `data/uploads/`
  - `data/generated/`

## Local Data Model

### `data/providers.json`

Stores provider metadata and encrypted API keys.

```json
{
  "providers": [
    {
      "id": "provider_uuid",
      "name": "packyapi",
      "baseUrl": "https://api.example.com",
      "encryptedApiKey": "...",
      "createdAt": "2026-06-14T00:00:00.000Z",
      "updatedAt": "2026-06-14T00:00:00.000Z"
    }
  ],
  "selectedProviderId": "provider_uuid"
}
```

### `data/uploads/uploads.json`

Stores program-managed input image copies.

```json
{
  "uploads": [
    {
      "id": "upload_uuid",
      "fileName": "image_1.png",
      "originalName": "source.png",
      "mimeType": "image/png",
      "size": 123456,
      "path": "data/uploads/image_1.png",
      "createdAt": "2026-06-14T00:00:00.000Z"
    }
  ],
  "nextIndex": 2
}
```

### `data/history.json`

Stores generated image records and retry metadata.

```json
{
  "items": [
    {
      "id": "image_uuid",
      "status": "succeeded",
      "mode": "text",
      "prompt": "A scene...",
      "providerId": "provider_uuid",
      "providerName": "packyapi",
      "model": "gpt-image-2",
      "size": "auto",
      "quality": "auto",
      "moderation": "auto",
      "sourceUrl": "https://...",
      "localFileName": "generated_uuid.png",
      "localPath": "data/generated/generated_uuid.png",
      "downloaded": false,
      "createdAt": "2026-06-14T00:00:00.000Z",
      "error": null,
      "retryPayload": {}
    }
  ]
}
```

## Backend API

### Providers

- `GET /api/providers`
- `POST /api/providers`
- `PUT /api/providers/:id`
- `DELETE /api/providers/:id`
- `POST /api/providers/select`

Frontend never receives the raw API key after save. Provider list returns masked API key metadata.

### Uploads

- `GET /api/uploads`
- `POST /api/uploads`
- `DELETE /api/uploads/:id`

Upload limit is 16. Uploaded files are copied to `data/uploads/` using sequential `image_N.ext` names.

### Generation

- `GET /api/history`
- `POST /api/generate`
- `POST /api/history/:id/retry`
- `POST /api/history/:id/downloaded`
- `GET /api/generated/:fileName`
- `GET /api/generated/:id/download`

Generation executes serially for count > 1. Each iteration creates a history record. On success, the server downloads the returned image URL into `data/generated/`. On failure, it stores a failed history record with the error message and retry payload.

## Startup Cleanup

At backend startup:

1. Read `data/history.json`.
2. Find generated image records where `downloaded` is false.
3. Delete their local files from `data/generated/`.
4. Remove or mark those records so stale unavailable previews do not appear.
5. Keep downloaded records and files.
6. Never delete files from `data/uploads/` during startup cleanup.

## UI Plan

### Left Sidebar

- Provider section:
  - selected provider dropdown
  - "Add provider" button
  - edit/delete controls for current provider
- Mode segmented control:
  - text-to-image
  - image-to-image
- Input images section visible and active for image-to-image:
  - upload dropzone/button
  - image cards with thumbnail, generated `image_N.ext` name, delete button
  - disabled/upload warning when 16-image limit is reached
- Parameter section:
  - model display fixed to `gpt-image-2`
  - size dropdown
  - quality dropdown
  - moderation switch
  - generation count input

### Main Area

- Top compact mode tabs matching the reference.
- Large preview area:
  - empty state when no result exists
  - loading state while generating
  - selected generated image preview
  - status pill such as `1/3`
- Custom right-click context menu on preview:
  - copy source URL
  - download image
  - copy local image path
- Bottom prompt composer:
  - multiline prompt input
  - submit button
  - disabled state while missing provider, prompt, or image-to-image inputs
  - inline error area

### Right Sidebar

- Add/new placeholder at top.
- Scrollable generated thumbnail list.
- Succeeded, failed, and loading states.
- Failed card includes retry action.
- Clicking thumbnail selects it in main preview.

## Acceptance Checklist

- App starts locally with one command.
- Provider can be added, selected, edited, and deleted.
- API key is encrypted in `providers.json`.
- Text-to-image request payload uses `gpt-image-2` and selected parameters.
- Image-to-image sends all uploaded input images.
- Uploads are renamed `image_N.ext`.
- Upload limit blocks the 17th image.
- Generation count > 1 sends serial requests.
- Errors are visible to the user and retryable.
- Generated history survives page refresh.
- Startup cleanup removes only not-downloaded generated files.
- Downloading from the page marks the image downloaded.
- Main image context menu supports source URL copy, local path copy, and download.
- Uploaded input image deletion does not touch the original user file.

## Known Initial Limits

- Only `gpt-image-2` is supported.
- Provider API must be OpenAI Images API compatible.
- `background` is fixed to `auto`.
- `output_format` is fixed to `png`.
- Generated count is serial and may be slow for large counts.
- Actual provider-specific image size support may differ; provider errors are shown directly.
